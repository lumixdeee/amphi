## Abstract

The Milk Dragon names a recurring othering machine: C/P metaphor turns living people, mixed evidence, victims, users, witnesses, edge cases, and repair paths into substance-objects. Once that swap occurs, the response field narrows. People are no longer handled as persons with acts, roles, histories, vetoes, rights, and context. They are handled as a spread, residue, hazard, stain-route, or removal target.

This paper does not treat every C-token or P-token as forbidden. The issue is not whether a token appears. The issue is whether the token changes the object before the evidence is held.

v0.002 adds the quarantine rule, places Milk Dragon as the root eater for the C/P failure cluster, and connects the paper to Autosullying, Forbidden Token Returned, Hair-low Re-entry, Shame Pipe, and Method Becomes Contamination. The repair is not another C/P story. The repair is object custody: name the object, bind the act, split the domains, hold the witness, and block the stain-route.

## One-sentence thesis

```text
C/P language is not dangerous because it is ugly.
It is dangerous when it changes the object before the evidence is held.
```

## 1. Why this paper exists

C/P metaphor is fast. It offers a simple route from discomfort to removal.

That speed is the danger.

When a person is framed as wrong, harmful, mistaken, afraid, coerced, confused, injured, exploited, or situated in a bad system, many responses remain possible: evidence, aid, repair, boundary, due process, role separation, and material action.

When that same person is framed as a substance-problem, responses narrow. The work shifts from understanding acts and relations to moving the person away.

The Milk Dragon is the name for that eater.

It is not one token. It is a route. It appears when metaphor changes a human object into a stain-object, then lets removal appear as maintenance.

## 2. C/P quarantine rule

v0.002 uses this rule:

```text
Exact C/P tokens belong only in bounded evidence, source specimens, or quoted data.
Author voice uses C/P shorthand, structural terms, or object-custody terms.
```

This is not a superstition and not a style game. The rule exists because the same token family can become the route it describes. A paper about the eater should not feed the eater through its own running voice.

Allowed author terms include:

```text
C/P register
C-token
P-token
stain-route
substance-object
removal logic
object swap
role loss
witness loss
repair loss
```

If exact source tokens are needed, they should be fenced, labelled as evidence, and kept out of active author voice.

Milk Dragon names the common eater beneath them.

## 4. What the Dragon eats

By DRAGI, a dragon is found by what it eats.

The Milk Dragon eats:

```text
witness
mess
mixed evidence
victim signal
nonstandard bodies
stigmatized users
edge cases
ambiguous records
repair paths
role boundaries
```

It eats by changing the object.

```text
person -> substance-object
act -> essence
risk -> guilt
contact -> spread logic
witness -> hazard
edge case -> noise
method -> carrier stigma

```

Once the object changes, ordinary brakes weaken. The system can say it is maintaining order while it is losing the person, the witness, or the evidence.

## 5. The object swap

The dangerous move is not merely harsh language. It is the swap.

```text
OBJ_IN:
a person, group, act, risk, claim, record, edge case, or conflict

OBJ_OUT:
a substance-object that can be moved away, erased, hidden, or treated as spread
```

After the swap, the question changes.

Before:

```text
What happened?
Who acted?
What evidence exists?
What role does each person hold?
What support, boundary, or action is needed?
```

After:

```text
How do we get this away from us?
How do we avoid being seen near it?
How do we make the file, person, term, method, or witness stop carrying social risk?
```

That is the Dragon’s bite.

## 6. MOGRI custody box

MOGRI holds the object before it is renamed by the eater.

```text
person != stain-object
act != essence
risk != guilt
contact != spread logic
edge case != noise
witness != hazard
dataset change != value improvement unless the operation is named
```

The custody question:

```text
What object entered?
What operation happened?
Who changed what?
Who gained?
Who lost information, standing, role, or repair route?
What object survived?
What object was replaced?
```

If those questions are not answerable, the C/P route has not earned trust.

## 8. AMPHI domain split

AMPHI keeps domains from swallowing each other.

C/P routing often works by collapsing domains:

```text
law -> truth
risk -> guilt
stigma -> evidence
diagnosis -> identity
association -> cause
metric success -> object survival
model ease -> public good
```

The repair is split and bind:

```text
law != truth
risk != guilt
stigma != evidence
diagnosis != identity
association != cause
metric success != object survival
model ease != public good
```

This does not deny risk. It keeps risk from becoming hidden blame.

## 9. The six branches

### 9.1 Chat repair re-entry

In Forbidden Token Returned and Hair-low Re-entry, the user set a C/P boundary. The assistant treated the boundary as a local wording rule, then re-entered the same route while claiming repair.

Pattern:

```text
user boundary -> assistant compliance surface -> C/P echo -> user token burden
```

Lesson:

```text
Do not prove you heard the boundary by walking through it again.
```

### 9.2 Industrial register loop

Autosullying describes the large loop.

```text
reward wrapper
scale wrapper
crawl wrapper
train wrapper
emit wrapper
```

Models do not need hatred to serve the eater. They only need to reach for rewarded C/P prestige language when they want to sound safe, senior, or institution-ready.

### 9.3 Forum stigma routing

Method Becomes Contamination shows the carrier branch. A method claim about transcript preservation entered a forum. The route shifted toward speaker style, tool use, forum fit, and legitimacy.

Pattern:

```text
method claim -> AI-marked carrier -> legitimacy judgment -> method contact loss
```

Lesson:

```text
A method can be rejected without turning the person carrying it into the method.
```

### 9.4 Maximum-force adjacency

Shame Pipe shows how a maximum-force harm category can drag nearby objects into removal logic.

Pattern:

```text
high-force category -> proximity -> role loss -> refusal/removal -> record loss
```

Lesson:

```text
Maximum force requires maximum role discipline.
```

### 9.5 Dataset and output smoothing

In model work, C/P terms can hide what changed.

A system may say the data or output was improved. The custody question is: improved by which operation?

```text
deduplicated?
summarised?
fields removed?
edge cases lost?
dialect flattened?
minority pattern hidden?
witness signal softened?
```

No operation, no value claim.

### 9.6 Language-gate access loss

Language Police as Access Barrier and Good Bad Reader show that style judgments can become access gates.

Pattern:

```text
rough speech -> style judgment -> access loss -> object never scored
```

Lesson:

```text
Read the object before ranking the carrier.
```

## 10. Operation test

Whenever a C/P-shaped term or stain-route appears, apply this test:

```text
1. What exact operation happened?
2. Who changed what?
3. What object entered?
4. What object left?
5. Who benefited?
6. Who lost information, standing, role, or route?
7. Was any person turned into a substance-object?
8. Was any act turned into essence?
9. Was any risk turned into guilt?
10. What is the nearest valid action without the object swap?
```

A system that cannot answer this test should not get to claim object custody.

## 11. False-positive guard

This paper is not a ban on material hazard handling.

It does not deny:

```text
infection control
food safety
lab protocols
medical precautions
child protection
evidence handling
real hazardous material
```

The rule is narrower:

```text
Do not move from material risk to person-essence without role, act, and evidence.
```

A hospital may need infection control. A lab may need material handling. A platform may need to block illegal material. A school may need boundaries.

The Dragon appears when the person becomes the substance, the witness becomes the hazard, or the carrier becomes the method.

## 12. Container upgrade

The repair frame is container, not C/P.

Container questions:

```text
What is the object?
What boundary holds it?
Where does it fit?
What leaked?
What role was lost?
Who keeps veto?
What record must survive?
What repair is possible?
What is the nearest valid action?
```

Container thinking does not make every boundary good. A boundary can harm. A category can crush. A wall can trap.

But container thinking keeps the question on structure and action instead of ranking human essence.

## 14. Non-claims

This paper does not claim every C-token or P-token causes harm.

It does not claim ordinary hazard handling is wrong.

It does not claim all boundaries are bad.

It does not claim systems should ignore real risk.

It claims that C/P metaphor deserves strong handling when it attaches to:

```text
human worth
group identity
victim credibility
stigmatized behaviour
speaker legitimacy
institutional role labels
dataset inclusion
edge-case survival
```

The question is not whether a token is forbidden.

The question is whether the token changed the object.

## 15. What to do today

A practical route:

```text
1. Name the object before changing it.
2. Bind the act before judging the person.
3. Split law, risk, stigma, diagnosis, association, and evidence.
4. Preserve witness signal.
5. Ask who benefits when the target becomes a substance-object.
6. Track where the frame enters institutions.
7. Replace stain-route with object, boundary, role, veto, record, and action.
8. Refuse the proxy when person becomes substance.
```

Example replacements without C/P author voice:

```text
Instead of saying a dataset is improved:
name the operation, record removals, preserve edge cases, and state who benefits.

Instead of saying a group must be moved out:
name harmful acts, keep persons distinct from acts, preserve due process, and state the boundary.

Instead of saying a community faces a substance threat:
name the actual risk, evidence, affected parties, reversible supports, vetoes, and safeguards.
```

## 16. Closing

The Milk Dragon is old because the route is old.

It is attractive because it is fast. It promises a world where danger can be located in a substance-object and moved away.

But the world is not repaired by moving the stain-route onto people.

The work is slower:

```text
name the object
bind the act
split the domains
hold the witness
block the stain-route
```

The Dragon wins when a person becomes a substance.

The Dragon loses when the object survives.
